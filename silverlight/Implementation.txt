The only requirements of a page that hosts this player are the silverlight object tag, which can be generated by the config tool, nad the OVP.xap file.


A very simple example page is show below, with a few additional tags for better formatting

<html>
<head>
    <title>OpenVideoPlayer Example</title>
    <style type="text/css">
        html, body {height: 100%;overflow: auto;}
        body {padding: 0; margin: 0;}
        #silverlightControlHost {height: 100%;}
    </style>
</head>

<body>
    <div id="silverlightControlHost">
		<object data="data:application/x-silverlight-2," type="application/x-silverlight-2" width="100%" height="100%" id="slp" >
		<param name="source" value="OVP.xap"/><param name="minRuntimeVersion" value="2.0.30923.0" /><param name="onerror" value="onSilverlightError" /><param name="MaxFrameRate" value="30" />
		<param name="initparams" value='showstats=true, autoplay=true, muted=false, playlistoverlay=true, stretchmode=Fit, feedsource=http%3A//rss.streamos.com/streamos/rss/genfeed.php%3Ffeedid%3D426%26groupname%3Dproducts' />
		<a href="http://go.microsoft.com/fwlink/?LinkID=124807" style="text-decoration: none;">
		<img src="http://go.microsoft.com/fwlink/?LinkId=108181" alt="Get Microsoft Silverlight" style="border-style: none"/></a>
		</object>
    </div>
</body>
</html>


*** Parameters ***

Most of the parameters are the same for any Silverlight app, and shouldn't be changed.  The only one that is relevant for us is the 'initparams' parameter, which takes name/value pairs that are parsed by the player to set it's initial state.  These are normally generated for you by the config tool  

The current options are:

    mediasource        - A direct absolute link to media content, http:// or mms://
                          Example: mediasource=http://openvideoplayer.com/test.wmv

    playlistsource     - A link to an Microsoft playlist xml, the format produced by Expression Encoder

    feedsource         - A link to a MediaRSS feed, must be HTTP or HTTPS, and a rss or xml file extension

    refsource          - A link to a reference file, supports asx, asf, wvx, wmv, wax, and wma

    playlist           - Directly provide the content of a playlist or reference file.
                          Example: playlist=<mediaDefinition><mediaItems><mediaItem mediaSource="sl.wmv"></mediaItem></mediaItems></mediaDefinition>

    muted              - true mutes the volume (default=false)

    autoplay           - true makes the player automatically start when the page is loaded (default=true)

    showstats          - true enables the statistics panel (default=false)

    playlistoverlay    - true enables overlay mode for the playlist (default=true)

    stretchmode        - Accepts the following modes by number, or either name:
                          0. Native  (None)
                          1. Fit     (Uniform)
                          2.         (Uniform to Fill) - not a direct match to any flash term
                          3. Stretch (Fill)

    showplaylist       - true to show playlist at start (default=false)

    showchapters       - true to show chapters box at start (default=false)

* Note, all bool params can be 1/0 or true/false
